Exploring Git

Git Commands Explained

git init: This command initializes a new Git repository in your current directory. It creates a hidden .git folder that contains all the necessary metadata for version control.

git add .: This command stages all the changes (new files, modified files, deleted files) in the current directory for the next commit. The dot (.) indicates that all changes should be added.

git commit -m "enter your meaningful message": This command saves the staged changes to the repository with a descriptive message. The message should summarize the changes made in that commit.

git branch -M main: This command renames the current branch to "main". The -M flag forces the rename even if the target branch name already exists.

git pull: This command fetches changes from a remote repository and merges them into your current branch. It is used to keep your local repository up-to-date with the remote repository.

git push: This command uploads your local commits to a remote repository. It is used to share your changes with others and update the remote repository with your latest work.

Most Commonly Used Git Commands

  1. git add . [To stage changes for the next commit]
  2. git commit -m "enter your meaningful message" [To save staged changes to the repository]
  3. git push [To upload local repository content to a remote repository]
  4. git pull [To fetch and integrate changes from a remote repository]
